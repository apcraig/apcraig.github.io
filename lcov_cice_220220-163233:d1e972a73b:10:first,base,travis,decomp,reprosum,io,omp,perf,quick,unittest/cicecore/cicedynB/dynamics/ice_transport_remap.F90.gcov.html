      61             : !=======================================================================
      62             : ! Here is some information about how the incremental remapping scheme
      63             : ! works in CICE and how it can be adapted for use in other models.  
      64             : !
      65             : ! The remapping routine is designed to transport a generic mass-like 
      66             : ! field (in CICE, the ice fractional area) along with an arbitrary number
      67             : ! of tracers in two dimensions.  The velocity components are assumed 
      68             : ! to lie at grid cell corners and the transported scalars at cell centers. 
      69             : ! Incremental remapping has the following desirable properties: 
      70             : ! 
      71             : ! (1) Tracer monotonicity is preserved.  That is, no new local 
      72             : !     extrema are produced in fields like ice thickness or internal 
      73             : !     energy. 
      74             : ! (2) The reconstucted mass and tracer fields vary linearly in x and y. 
      75             : !     This means that remapping is 2nd-order accurate in space, 
      76             : !     except where horizontal gradients are limited to preserve 
      77             : !     monotonicity. 
      78             : ! (3) There are economies of scale.  Transporting a single field 
      79             : !     is rather expensive, but additional fields have a relatively 
      80             : !     low marginal cost. 
      81             : ! 
      82             : ! The following generic conservation equations may be solved: 
      83             : ! 
      84             : !            dm/dt = del*(u*m)             (0) 
      85             : !       d(m*T1)/dt = del*(u*m*T1)          (1) 
      86             : !    d(m*T1*T2)/dt = del*(u*m*T1*T2)       (2) 
      87             : ! d(m*T1*T2*T3)/dt = del*(u*m*T1*T2*T3)    (3) 
      88             : !
      89             : ! where d is a partial derivative, del is the 2D divergence operator,
      90             : ! u is the horizontal velocity, m is the mass density field, and
      91             : ! T1, T2, and T3 are tracers.
      92             : !
      93             : ! In CICE, these equations have the form
      94             : ! 
      95             : !               da/dt = del*(u*a)          (4)
      96             : ! dv/dt =   d(a*h)/dt = del*(u*a*h)        (5)
      97             : ! de/dt = d(a*h*q)/dt = del*(u*a*h*q)      (6)
      98             : !            d(aT)/dt = del*(u*a*t)        (7)
      99             : ! 
     100             : ! where a = fractional ice area, v = ice/snow volume, h = v/a = thickness, 
     101             : ! e = ice/snow internal energy (J/m^2), q = e/v = internal energy per 
     102             : ! unit volume (J/m^3), and T is a tracer.  These equations express 
     103             : ! conservation of ice area, volume, internal energy, and area-weighted
     104             : ! tracer, respectively. 
     105             : !
     106             : ! (Note: In CICE, a, v and e are prognostic quantities from which
     107             : !  h and q are diagnosed.  The remapping routine works with tracers,
     108             : !  which means that h and q must be derived from a, v, and e before
     109             : !  calling the remapping routine.)  
     110             : !
     111             : ! Earlier versions of CICE assumed fixed ice and snow density. 
     112             : ! Beginning with CICE 4.0, the ice and snow density can be variable. 
     113             : ! In this case, equations (5) and (6) are replaced by 
     114             : ! 
     115             : ! dv/dt =        d(a*h)/dt = del*(u*a*h)          (8)  
     116             : ! dm/dt =    d(a*h*rho)/dt = del*(u*a*h*rho)      (9)
     117             : ! de/dt = d(a*h*rho*qm)/dt = del*(u*a*h*rho*qm)   (10)
     118             : ! 
     119             : ! where rho = density and qm = internal energy per unit mass (J/kg). 
     120             : ! Eq. (9) expresses mass conservation, which in the variable-density 
     121             : ! case is no longer equivalent to volume conservation (8). 
     122             : !
     123             : ! Tracers satisfying equations of the form (1) are called "type 1." 
     124             : ! In CICE the paradigmatic type 1 tracers are hi and hs. 
     125             : ! 
     126             : ! Tracers satisfying equations of the form (2) are called "type 2". 
     127             : ! The paradigmatic type 2 tracers are qi and qs (or rhoi and rhos 
     128             : !  in the variable-density case). 
     129             : ! 
     130             : ! Tracers satisfying equations of the form (3) are called "type 3."
     131             : ! The paradigmatic type 3 tracers are qmi and qms in the variable-density
     132             : ! case.  There are no such tracers in the constant-density case. 
     133             : ! 
     134             : ! The fields a, T1, and T2 are reconstructed in each grid cell with 
     135             : ! 2nd-order accuracy.  T3 is reconstructed with 1st-order accuracy 
     136             : ! (i.e., it is transported in upwind fashion) in order to avoid 
     137             : ! additional mathematical complexity. 
     138             : ! 
     139             : ! The mass-like field lives in the array "mm" (shorthand for mean 
     140             : ! mass) and the tracers fields in the array "tm" (mean tracers). 
     141             : ! In order to transport tracers correctly, the remapping routine 
     142             : ! needs to know the tracers types and relationships.  This is done 
     143             : ! as follows: 
     144             : ! 
     145             : ! Each field in the "tm" array is assigned an index, 1:ntrace. 
     146             : ! (Note: ntrace is not the same as ntrcr, the number of tracers 
     147             : ! in the trcrn state variable array.  For remapping purposes we 
     148             : ! have additional tracers hi and hs.) 
     149             : ! 
     150             : ! The tracer types (1,2,3) are contained in the "tracer_type" array. 
     151             : ! For standard CICE: 
     152             : ! 
     153             : !     tracer_type = (1 1 1 2 2 2 2 2) 
     154             : ! 
     155             : ! Type 2 and type 3 tracers are said to depend on type 1 tracers. 
     156             : ! For instance, qi depends on hi, which is to say that 
     157             : ! there is a conservation equation of the form (2) or (6). 
     158             : ! Thus we define a "depend" array.  For standard CICE: 
     159             : ! 
     160             : !          depend = (0 0 0 1 1 1 1 2) 
     161             : ! 
     162             : ! which implies that elements 1-3 (hi, hs, Ts) are type 1, 
     163             : ! elements 4-7 (qi) depend on element 1 (hi), and element 8 (qs) 
     164             : ! depends on element 2 (hs). 
     165             : !
     166             : ! We also define a logical array "has_dependents".  In standard CICE: 
     167             : ! 
     168             : !  has_dependents = (T T F F F F F F), 
     169             : ! 
     170             : ! which means that only elements 1 and 2 (hi and hs) have dependent 
     171             : ! tracers. 
     172             : ! 
     173             : ! For the variable-density case, things are a bit more complicated. 
     174             : ! Suppose we have 4 variable-density ice layers and one variable- 
     175             : ! density snow layer.  Then the indexing is as follows: 
     176             : ! 1    = hi 
     177             : ! 2    = hs 
     178             : ! 3    = Ts 
     179             : ! 4-7  = rhoi 
     180             : ! 8    = rhos 
     181             : ! 9-12 = qmi 
     182             : ! 13   = qms 
     183             : ! 
     184             : ! The key arrays are: 
     185             : ! 
     186             : !    tracer_type = (1 1 1 2 2 2 2 2 3 3 3 3 3) 
     187             : ! 
     188             : !         depend = (0 0 0 1 1 1 1 2 4 5 6 7 8) 
     189             : ! 
     190             : ! has_dependents = (T T F T T T T T F F F F F) 
     191             : ! 
     192             : ! which imply that hi and hs are type 1 with dependents rhoi and rhos, 
     193             : ! while rhoi and rhos are type 2 with dependents qmi and qms. 
     194             : ! 
     195             : ! Tracers added to the ntrcr array are handled automatically 
     196             : ! by the remapping with little extra coding.  It is necessary 
     197             : ! only to provide the correct type and dependency information. 
     198             : !
     199             : ! When using this routine in other models, most of the tracer dependency
     200             : ! apparatus may be irrelevant.  In a layered ocean model, for example,
     201             : ! the transported fields are the layer thickness h (the mass density
     202             : ! field) and two or more tracers (T, S, and various trace species).
     203             : ! Suppose there are just two tracers, T and S.  Then the tracer arrays
     204             : ! have the values:
     205             : !
     206             : !    tracer_type = (1 1)
     207             : !         depend = (0 0)
     208             : ! has_dependents = (F F)
     209             : !
     210             : ! which is to say that all tracer transport equations are of the form (1).
     211             : !
     212             : ! The tracer dependency arrays are optional input arguments for the
     213             : ! main remapping subroutine.  If these arrays are not passed in, they
     214             : ! take on the default values tracer_type(:) = 1, depend(:) = 0, and
     215             : ! has_dependents(:) = F, which are appropriate for most purposes.
     216             : !
     217             : ! Another optional argument is integral_order.  If integral_order = 1,
     218             : ! then the triangle integrals are exact for linear functions of x and y.
     219             : ! If integral_order = 2, these integrals are exact for both linear and
     220             : ! quadratic functions.  If integral_order = 3, integrals are exact for
     221             : ! cubic functions as well.  If all tracers are of type 1, then the
     222             : ! integrals of mass*tracer are quadratic, and integral_order = 2 is
     223             : ! sufficient.  In CICE, where there are type 2 tracers, we integrate
     224             : ! functions of the form mass*tracer1*tracer2.  Thus integral_order = 3
     225             : ! is required for exactness, though integral_order = 2 may be good enough
     226             : ! in practice.
     227             : !
     228             : ! Finally, a few words about the edgearea fields:
     229             : !
     230             : ! In earlier versions of this scheme, the divergence of the velocity
     231             : ! field implied by the remapping was, in general, different from the
     232             : ! value of del*u computed in the dynamics.  For energetic consistency
     233             : ! (in CICE as well as in layered ocean models such as HYPOP),
     234             : ! these two values should agree.  This can be ensured by setting
     235             : ! l_fixed_area = T and specifying the area transported across each grid
     236             : ! cell edge in the arrays edgearea_e and edgearea_n.  The departure
     237             : ! regions are then tweaked, following an idea by Mats Bentsen, such
     238             : ! that they have the desired area.  If l_fixed_area = F, these regions
     239             : ! are not tweaked, and the edgearea arrays are output variables.
     240             : !
