      69              : !=======================================================================
      70              : ! Here is some information about how the incremental remapping scheme
      71              : ! works in CICE and how it can be adapted for use in other models.
      72              : !
      73              : ! The remapping routine is designed to transport a generic mass-like
      74              : ! field (in CICE, the ice fractional area) along with an arbitrary number
      75              : ! of tracers in two dimensions.  The velocity components are assumed
      76              : ! to lie at grid cell corners and the transported scalars at cell centers.
      77              : ! Incremental remapping has the following desirable properties:
      78              : !
      79              : ! (1) Tracer monotonicity is preserved.  That is, no new local
      80              : !     extrema are produced in fields like ice thickness or internal
      81              : !     energy.
      82              : ! (2) The reconstucted mass and tracer fields vary linearly in x and y.
      83              : !     This means that remapping is 2nd-order accurate in space,
      84              : !     except where horizontal gradients are limited to preserve
      85              : !     monotonicity.
      86              : ! (3) There are economies of scale.  Transporting a single field
      87              : !     is rather expensive, but additional fields have a relatively
      88              : !     low marginal cost.
      89              : !
      90              : ! The following generic conservation equations may be solved:
      91              : !
      92              : !            dm/dt = del*(u*m)             (0)
      93              : !       d(m*T1)/dt = del*(u*m*T1)          (1)
      94              : !    d(m*T1*T2)/dt = del*(u*m*T1*T2)       (2)
      95              : ! d(m*T1*T2*T3)/dt = del*(u*m*T1*T2*T3)    (3)
      96              : !
      97              : ! where d is a partial derivative, del is the 2D divergence operator,
      98              : ! u is the horizontal velocity, m is the mass density field, and
      99              : ! T1, T2, and T3 are tracers.
     100              : !
     101              : ! In CICE, these equations have the form
     102              : !
     103              : !               da/dt = del*(u*a)          (4)
     104              : ! dv/dt =   d(a*h)/dt = del*(u*a*h)        (5)
     105              : ! de/dt = d(a*h*q)/dt = del*(u*a*h*q)      (6)
     106              : !            d(aT)/dt = del*(u*a*t)        (7)
     107              : !
     108              : ! where a = fractional ice area, v = ice/snow volume, h = v/a = thickness,
     109              : ! e = ice/snow internal energy (J/m^2), q = e/v = internal energy per
     110              : ! unit volume (J/m^3), and T is a tracer.  These equations express
     111              : ! conservation of ice area, volume, internal energy, and area-weighted
     112              : ! tracer, respectively.
     113              : !
     114              : ! (Note: In CICE, a, v and e are prognostic quantities from which
     115              : !  h and q are diagnosed.  The remapping routine works with tracers,
     116              : !  which means that h and q must be derived from a, v, and e before
     117              : !  calling the remapping routine.)
     118              : !
     119              : ! Earlier versions of CICE assumed fixed ice and snow density.
     120              : ! Beginning with CICE 4.0, the ice and snow density can be variable.
     121              : ! In this case, equations (5) and (6) are replaced by
     122              : !
     123              : ! dv/dt =        d(a*h)/dt = del*(u*a*h)          (8)
     124              : ! dm/dt =    d(a*h*rho)/dt = del*(u*a*h*rho)      (9)
     125              : ! de/dt = d(a*h*rho*qm)/dt = del*(u*a*h*rho*qm)   (10)
     126              : !
     127              : ! where rho = density and qm = internal energy per unit mass (J/kg).
     128              : ! Eq. (9) expresses mass conservation, which in the variable-density
     129              : ! case is no longer equivalent to volume conservation (8).
     130              : !
     131              : ! Tracers satisfying equations of the form (1) are called "type 1."
     132              : ! In CICE the paradigmatic type 1 tracers are hi and hs.
     133              : !
     134              : ! Tracers satisfying equations of the form (2) are called "type 2".
     135              : ! The paradigmatic type 2 tracers are qi and qs (or rhoi and rhos
     136              : !  in the variable-density case).
     137              : !
     138              : ! Tracers satisfying equations of the form (3) are called "type 3."
     139              : ! The paradigmatic type 3 tracers are qmi and qms in the variable-density
     140              : ! case.  There are no such tracers in the constant-density case.
     141              : !
     142              : ! The fields a, T1, and T2 are reconstructed in each grid cell with
     143              : ! 2nd-order accuracy.  T3 is reconstructed with 1st-order accuracy
     144              : ! (i.e., it is transported in upwind fashion) in order to avoid
     145              : ! additional mathematical complexity.
     146              : !
     147              : ! The mass-like field lives in the array "mm" (shorthand for mean
     148              : ! mass) and the tracers fields in the array "tm" (mean tracers).
     149              : ! In order to transport tracers correctly, the remapping routine
     150              : ! needs to know the tracers types and relationships.  This is done
     151              : ! as follows:
     152              : !
     153              : ! Each field in the "tm" array is assigned an index, 1:ntrace.
     154              : ! (Note: ntrace is not the same as ntrcr, the number of tracers
     155              : ! in the trcrn state variable array.  For remapping purposes we
     156              : ! have additional tracers hi and hs.)
     157              : !
     158              : ! The tracer types (1,2,3) are contained in the "tracer_type" array.
     159              : ! For standard CICE:
     160              : !
     161              : !     tracer_type = (1 1 1 2 2 2 2 2)
     162              : !
     163              : ! Type 2 and type 3 tracers are said to depend on type 1 tracers.
     164              : ! For instance, qi depends on hi, which is to say that
     165              : ! there is a conservation equation of the form (2) or (6).
     166              : ! Thus we define a "depend" array.  For standard CICE:
     167              : !
     168              : !          depend = (0 0 0 1 1 1 1 2)
     169              : !
     170              : ! which implies that elements 1-3 (hi, hs, Ts) are type 1,
     171              : ! elements 4-7 (qi) depend on element 1 (hi), and element 8 (qs)
     172              : ! depends on element 2 (hs).
     173              : !
     174              : ! We also define a logical array "has_dependents".  In standard CICE:
     175              : !
     176              : !  has_dependents = (T T F F F F F F),
     177              : !
     178              : ! which means that only elements 1 and 2 (hi and hs) have dependent
     179              : ! tracers.
     180              : !
     181              : ! For the variable-density case, things are a bit more complicated.
     182              : ! Suppose we have 4 variable-density ice layers and one variable-
     183              : ! density snow layer.  Then the indexing is as follows:
     184              : ! 1    = hi
     185              : ! 2    = hs
     186              : ! 3    = Ts
     187              : ! 4-7  = rhoi
     188              : ! 8    = rhos
     189              : ! 9-12 = qmi
     190              : ! 13   = qms
     191              : !
     192              : ! The key arrays are:
     193              : !
     194              : !    tracer_type = (1 1 1 2 2 2 2 2 3 3 3 3 3)
     195              : !
     196              : !         depend = (0 0 0 1 1 1 1 2 4 5 6 7 8)
     197              : !
     198              : ! has_dependents = (T T F T T T T T F F F F F)
     199              : !
     200              : ! which imply that hi and hs are type 1 with dependents rhoi and rhos,
     201              : ! while rhoi and rhos are type 2 with dependents qmi and qms.
     202              : !
     203              : ! Tracers added to the ntrcr array are handled automatically
     204              : ! by the remapping with little extra coding.  It is necessary
     205              : ! only to provide the correct type and dependency information.
     206              : !
     207              : ! When using this routine in other models, most of the tracer dependency
     208              : ! apparatus may be irrelevant.  In a layered ocean model, for example,
     209              : ! the transported fields are the layer thickness h (the mass density
     210              : ! field) and two or more tracers (T, S, and various trace species).
     211              : ! Suppose there are just two tracers, T and S.  Then the tracer arrays
     212              : ! have the values:
     213              : !
     214              : !    tracer_type = (1 1)
     215              : !         depend = (0 0)
     216              : ! has_dependents = (F F)
     217              : !
     218              : ! which is to say that all tracer transport equations are of the form (1).
     219              : !
     220              : ! The tracer dependency arrays are optional input arguments for the
     221              : ! main remapping subroutine.  If these arrays are not passed in, they
     222              : ! take on the default values tracer_type(:) = 1, depend(:) = 0, and
     223              : ! has_dependents(:) = F, which are appropriate for most purposes.
     224              : !
     225              : ! Another optional argument is integral_order.  If integral_order = 1,
     226              : ! then the triangle integrals are exact for linear functions of x and y.
     227              : ! If integral_order = 2, these integrals are exact for both linear and
     228              : ! quadratic functions.  If integral_order = 3, integrals are exact for
     229              : ! cubic functions as well.  If all tracers are of type 1, then the
     230              : ! integrals of mass*tracer are quadratic, and integral_order = 2 is
     231              : ! sufficient.  In CICE, where there are type 2 tracers, we integrate
     232              : ! functions of the form mass*tracer1*tracer2.  Thus integral_order = 3
     233              : ! is required for exactness, though integral_order = 2 may be good enough
     234              : ! in practice.
     235              : !
     236              : ! Finally, a few words about the edgearea fields:
     237              : !
     238              : ! In earlier versions of this scheme, the divergence of the velocity
     239              : ! field implied by the remapping was, in general, different from the
     240              : ! value of del*u computed in the dynamics.  For energetic consistency
     241              : ! (in CICE as well as in layered ocean models such as HYPOP),
     242              : ! these two values should agree.  This can be ensured by setting
     243              : ! l_fixed_area = T and specifying the area transported across each grid
     244              : ! cell edge in the arrays edgearea_e and edgearea_n.  The departure
     245              : ! regions are then tweaked, following an idea by Mats Bentsen, such
     246              : ! that they have the desired area.  If l_fixed_area = F, these regions
     247              : ! are not tweaked, and the edgearea arrays are output variables.
